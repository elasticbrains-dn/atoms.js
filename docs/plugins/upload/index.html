AtomUpload Plugin

This plugin provides Flash fallback as well as it provides Pre Post to securely create non cookie based upload URL.

Properties

upload-url

When using non-flash mode, you can simply set upload-url to HTTP URL where files will be uploaded.

upload-on-select

By default it is true, files are uploaded as soon as they are selected.

post-url

When using pre-post method, a JSON payload will be uploaded to specified post-url along with specified post-data. The result must contain a valid upload URL at upload-path specified.

post-data

When using pre-post method, this property must contain JSON payload data. It should not be JSON encoded, it can be any JavaScript object type.

upload-path

When using pre-post method, upload-path will be used to retrive upload url from the resultant JSON data.

Commands

uploadCommand

If upload-on-select is false, this command must be specified as next to any UI operation or any form post.

Modes of Usage

Pre Upload Post

Since flash does not send cookies along with upload, it is recommended that first we post data and server must send a unique one time usage upload url.

Upload Url

If you are ok with not sending cookies for upload validation, then you can directly bind upload-url property where the files will be uploaded.
